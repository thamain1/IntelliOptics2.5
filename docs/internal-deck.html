Quality Items Shipped

Seven Detection Quality Fixes

1–2

Per-class thresholds at edge + model version stamping on escalations

Drift endpoint at GET /detectors/{id}/drift — per-detector threshold calibration applied at inference time, escalations carry model version for auditability.

3

detect_tiled() — 2×2 grid with cross-tile NMS + low-conf VLM fallback

Tiled inference improves small-object recall. Results merged with NMS across tile boundaries. VLM fallback fires when max_conf < 0.25.

4

Escalation queue priority — filename prefix sort

p{nn}- filename prefix, sorted by QueueReader — high-confidence detections escalate ahead of ambiguous ones.

5

Temporal deduplication — IoU-based within 2s window

IoU ≥ 0.7 within 2-second window prevents alert flooding from repeated detections of the same object across frames.

6

OODD per-detector calibrated threshold

ResNet18-based out-of-domain detector now uses per-detector calibrated threshold — replaces hardcoded 0.444 default.

7

Letterbox padding 114 → 0

Black padding (value 0) matches ImageNet training distribution. Previous gray padding (114) caused subtle inference drift on padded regions.

Active Learning

The Full Learning Loop

Six phases. Closed loop from labeled data to deployed model — automatically.

Phase 1 — Dataset Export
80/20 train/val split, YOLO format zip
Minimum 50 samples required
Uploads to Supabase models bucket
Phase 2 — YOLO Fine-Tuning
cloud-trainer runs ultralytics
Auto-detects GPU (CUDA 7.0+)
Exports ONNX candidate
Updates training_runs via PostgREST
Phase 3 — Canary Shadow
Candidate runs alongside primary on every live query
Results saved to shadow_detections
Agreement rate tracked over time
Phase 4+5 — Promote / Rollback
Promote candidate to primary
One-level rollback via previous_primary_model_path
Edge picks up new model in ≤ 60s
Phase 6 — Auto-Trigger
APScheduler 1h check
Fires when ≥ 100 new labeled samples
SendGrid email on trigger + completion

Architecture Decisions

Why We Built It This Way

Decision

ultralytics in a separate cloud-trainer container

Rationale

AGPL-3.0 never touches the Apache 2.0 inference path. cloud-worker uses pure ONNX Runtime — safe to ship commercially without license concerns.

Decision

Trainer writes via PostgREST, not direct Supabase client

Rationale

Background workers don't have a user JWT session. PostgREST accepts the service key and handles writes cleanly without callback auth complexity.

Decision

Shadow mode lives in cloud-worker, not edge-api

Rationale

inference_is_available() always returns False in Docker — K8s hostnames don't exist. All inference routes through cloud-worker anyway, making it the natural host for canary logic.

Decision

Normalize bboxes 0–1 xyxy at worker output

Rationale

Frontend overlays work correctly without needing original image dimensions. Canvas scaling just multiplies by container dimensions — no coordinate conversion per-frame.

Decision

APScheduler in-process (not a separate container)

Rationale

No extra container to manage. Survives backend restarts. next_run_time offset avoids cold-start triggering on first boot.

Decision

Letterbox padding value: 0 (black), not 114 (gray)

Rationale

Matches ImageNet training distribution. Gray padding (114) caused subtle inference drift on padded border regions — detections near frame edges were less reliable.
